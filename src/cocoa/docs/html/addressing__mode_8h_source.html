7//
8// \file addressing_mode.h
9//
10// \brief CPU addressing modes
11//
12// Inherent addressing
13// Inherent addressing is used when the target
14// operand is inherently specified in the instruction
15// itself. For example, the CLRA instruction specifies
16// to clear the A register.
17//
18// Immediate addressing
19// Similar to extended addressing, instructions that
20// use this addressing mode specify their target as
21// either an 8-bit or 16-bit constant value that is
22// encoded in the instruction itself. "Immediate"
23// implies the constant is used as DATA. For example,
24// ADDA #$30 adds 0x30 to A.
25//
26// Extended addressing
27// Similar to immediate addressing, instructions that
28// use this addressing mode specify their target as
29// either a 16-bit constant value that is encoded
30// in the instruction itself. "Extended" implies
31// the constant is used as an ADDRESS (and thus
32// is always 16-bit). For example, JSR $1234 jumps
33// to address 0x1234 as a subroutine call.
34//
35// Extended indirect addressing
36// This mode is similar to extended addressing,
37// except that the 16-bit constant encoded into the
38// instruction is used as the ADDRESS OF AN ADDRESS.
39// For example, JSR [$1234] (where the square braces
40// indicate extended indirect addressing) would jump
41// to the address formed by the 16-bit word at memory
42// locations $1234 and $1235.
43//
44// Indexed
45// Instructions that use this mode have two operands:
46// an index register and EITHER a 5-bit constant or
47// an 8-bit register name. When the two operands are
48// added together, the result forms the address on
49// which the instruction acts. For example, STA 2,X
50// stores the A register in the memory location (2+X),
51// and STA B,X stores the A register in the memory
52// location (B+X). Note that the addition is always
53// UNSIGNED. Indexed addressing is implied by use
54// of the X index register as an operand.
55//
56// Indexed indirect addressing
57// This mode combines the address-specification
58// ability of indexed addressing with the address-of-
59// address ability of extended indirect addressing.
60// For example, STA [2,X] (where the square braces
61// indicate indirect addressing, and use of the X
62// index register indicates indexed addressing) stores
63// the A register into the 16-bit memory location formed
64// from reading memory locations (2+X) and (2+X+1).
65//
66// Relative addressing
67// In this mode, the constant operand is treated as
68// either an 8-bit SIGNED or 16-bit SIGNED offset
69// from the PCR (program counter register, instruction
70// pointer, whatever you want to call it). Note that
71// the PCR is moved to the next instruction AFTER
72// evaluation of this value, so that if the instruction
73// BRA $0585 is at memory location $0580, this assembles
74// to a 16-bit instruction whose relative offset is +5.
75// For assembly programmers, the only relevance here is
76// that BRA is a 16-bit instruction whose offset is 8-bit
77// and has range -128...+127, whereas LBRA is a 24-bit
78// instruction whose offset is 16-bit and has range
79// -32768...+32767.
80//
81// Direct
82// In general, instructions that use Extended addressing
83// also support Direct addressing. Direct addressing
84// allows the programmer to specify only the LOWER 8
85// bits of the address; the UPPER 8 bits are taken from
86// the DP (direct page) register. To specify this mode,
87// the left angle bracket is used: JSR <$15 will form
88// the target jump address by using the DP register as
89// the upper 8-bits and $15 as the lower 8 bits.
90//
